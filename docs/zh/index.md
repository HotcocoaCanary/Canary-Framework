# Canary Framework

Canary Framework 是一个轻量级、面向切面、纯装饰器驱动的 Python 异步基础设施框架。
不同于传统的 Web 框架，Canary 的核心灵魂是一个**纯粹的依赖注入（DI）和生命周期容器**，专为构建复杂、高扩展性、模块化的应用程序（如智能体 Agent 系统、复杂的 Web 服务后台等）而设计。

## 核心特性

- **纯装饰器架构**：无需继承任何基类，只需 `@service()` 或 `@module()` 即可将任何普通类纳入框架管理。
- **极致的依赖注入 (DI)**：全自动的、基于 Python 类型注解（Type Hints）的依赖注入机制。零侵入，零显式调用。
- **Kahn 拓扑排序**：引擎在启动时会通过图论算法计算出无懈可击的服务启动顺序，彻底避免循环依赖死锁。
- **微内核与插件化**：框架的核心仅仅负责依赖和生命周期（`init`, `startup`, `shutdown`）。如 HTTP 路由（Router）、OpenAPI 等能力全部作为 `core/web/` 插件包隔离。
- **Flat 路由系统**：虽然核心是 DI 容器，但内置了原生极简的 Starlette HTTP 路由解析器，具备极速的性能和自动化的 OpenAPI 3.0.3 文档生成。
