# FastAPI Integration

The `canary_framework.web.fastapi` subpackage provides `@router`, HTTP method decorators, and `WebCanary` for FastAPI + Uvicorn integration.

## Minimal Web Application

A single `@router` class passed directly to `WebCanary`:

```python
from canary_framework.web.fastapi import WebCanary, router, get

@router(prefix="/")
class RootRouter:
    @get("/health")
    async def health(self) -> dict[str, str]:
        return {"status": "ok"}

app = WebCanary(RootRouter)
await app.init()
await app.start()  # starts Uvicorn on 127.0.0.1:8000
```

## Full Example: Router with Dependencies

```python
from canary_framework import module, service, on_init
from canary_framework.web.fastapi import WebCanary, router, get, post

@service(name="user_service")
class UserService:
    users: list[dict] = []

    def list_users(self) -> list[dict]:
        return self.users

    def create_user(self, name: str) -> dict:
        user = {"name": name}
        self.users.append(user)
        return user

@router(prefix="/api/users", deps=[UserService], tags=["users"])
class UserRouter:
    user_service: UserService

    @get("/")
    async def list_users(self) -> list[dict]:
        return self.user_service.list_users()

    @post("/", status_code=201)
    async def create_user(self, body: dict) -> dict:
        return self.user_service.create_user(body["name"])

@module("app", services=[UserService, UserRouter])
class AppModule:
    pass

app = WebCanary(AppModule)
await app.init()
await app.start()
```

## HTTP Method Decorators

| Decorator | HTTP Method | Example |
|-----------|-------------|---------|
| `@get(path)` | GET | `@get("/users/{id}")` |
| `@post(path)` | POST | `@post("/users", status_code=201)` |
| `@put(path)` | PUT | `@put("/users/{id}")` |
| `@delete(path)` | DELETE | `@delete("/users/{id}")` |
| `@patch(path)` | PATCH | `@patch("/users/{id}")` |

Each decorator accepts these optional keyword arguments:

`response_model`, `status_code`, `summary`, `description`, `tags`, `dependencies`, `deprecated`, `response_description`

These map directly to FastAPI's route parameters — no need to learn a different API.

## Router Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `prefix` | `str` | URL prefix applied to all routes in this group |
| `name` | `str` | Service name (auto-generated via `to_snake` if omitted) |
| `deps` | `list[type]` | Dependencies injected via DI |
| `tags` | `list[str]` | OpenAPI tags applied to all routes |

## Configuration Prefixes

Root config model fields with `uvicorn_` or `fastapi_` prefixes are automatically routed:

```python
from pydantic import BaseModel

class AppConfig(BaseModel):
    uvicorn_host: str = "127.0.0.1"
    uvicorn_port: int = 8000
    fastapi_title: str = "My API"
    database_url: str = "postgresql://"  # no prefix → business config
```

## HTTP Request Logging

`WebCanary` configures Uvicorn's access log with the CF log format:

```
[CF] [INFO ] [cf.web] 127.0.0.1:52341 - "GET /api/users HTTP/1.1" 200
[CF] [INFO ] [cf.web] 127.0.0.1:52342 - "POST /api/users HTTP/1.1" 201
```

Zero overhead — logs are generated by Uvicorn at the protocol layer.
